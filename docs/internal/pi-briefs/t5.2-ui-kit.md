## Goal

Build the following primitives in `frontend/src/components/ui/` (new directory), one file each,
default-exported component, PLUS a barrel file `frontend/src/components/ui/index.ts` that
re-exports everything as named exports. Do not touch any other existing component or screen —

### 1. `Button.tsx`
Props: `variant?: "primary" | "secondary" | "ghost" | "destructive"` (default `"primary"`),
`size?: "sm" | "md"` (default `"md"`), `loading?: boolean`, plus all native
`ButtonHTMLAttributes<HTMLButtonElement>` (so `onClick`, `disabled`, `type`, etc. pass through
and `children` is the label/content).

- `primary`: `bg-accent` fill, white text, hover: `bg-accent-hover` + a 1px `-translate-y-px`
  lift, transition on `duration-hover`.
- `secondary`: `bg-surface-2` fill, `border border-border` (hairline), `text-text`.
- `ghost`: transparent, `text-text-dim`, hover `bg-surface-2` + `text-text`.
- `destructive`: `bg-gap` fill, white text (same hover-lift treatment as primary).
- Sizes: `sm` = smaller padding/text (e.g. `px-2.5 py-1 text-xs`), `md` = default
  (e.g. `px-4 py-2 text-sm`). Both `rounded-control`.
- **Loading state**: when `loading` is true — button becomes `disabled` and gets
  `aria-busy="true"`. The label/children stay visible (do NOT replace children with just a
  spinner — that collapses the button width). Render a 14px spinner (a simple CSS-animated SVG
  or div — no new dependency needed) to the LEFT of the children, inside the button, so the
  button grows slightly rather than swapping content.
- Every state must remain reachable via keyboard focus (visible focus ring using `accent` at
  low opacity — e.g. `focus-visible:ring-2 focus-visible:ring-accent/40`).

### 2. `Input.tsx`, `Textarea.tsx`
Both: `bg-surface` fill, `border border-border`, `rounded-control`, `text-text` value text,
placeholder in `text-text-faint`. Focus state: `border-border-strong` + a 2px ring at 40% alpha
of `--accent` (Tailwind: `focus:ring-2 focus:ring-accent/40 focus:border-border-strong`,
remove default outline). Forward all native `InputHTMLAttributes`/`TextareaHTMLAttributes` and
`ref` (use `forwardRef`).

Also build a shared `Label.tsx` (12px, uppercase, `text-text-dim`, `tracking-wide`) and a
`FieldError.tsx` (12px, `text-gap`, fades in over `duration-hover` — use a simple CSS
`transition-opacity` mount, framer-motion is not required for this one). These three exist so
callers can compose `<Label>…</Label><Input/><FieldError>…</FieldError>` — don't build a
combined "FormField" wrapper, keep them separate primitives.

### 3. `Select.tsx`
Wrap `@radix-ui/react-select`. Props: `value`, `onValueChange`, `options: {value: string; label:
string}[]`, `placeholder?: string`, plus optional `disabled`. Trigger styled like `Input`
(`bg-surface`, `border-border`, `rounded-control`, same focus ring treatment). Content/viewport:
`bg-surface-2`, `border-border`, `rounded-card`, `shadow-elevated`, items highlight with
`bg-surface` on hover/focus. Include the chevron icon from `lucide-react`
(`ChevronDown`) in the trigger.

### 4. `Pill.tsx` (StatusPill)
Props: `status: string` (one of the known run/case statuses) or, more simply, accept
`tone: "run" | "partial" | "met" | "gap" | "dim"` directly plus `label: string` — **implement it
this way** (tone-based, not a hardcoded status→tone map) so callers decide the mapping; don't
try to encode the full status list from the plan (intake/analyzing/drafting/etc.) inside this
component, that mapping belongs to the screens that use it later (T5.5/T5.6/T5.7).

`bg-run/10` — Tailwind's `/10` opacity modifier works since these are CSS vars wired through

used elsewhere), text/dot color = the full tone color, `font-mono text-[11px] uppercase`

### 5. Toast system: `Toast.tsx` + `ToastProvider.tsx` + a `useToast` hook
Wrap `@radix-ui/react-toast`. Build:
- `ToastProvider.tsx`: wraps Radix's `Toast.Provider` + `Toast.Viewport` (positioned
  bottom-right, `fixed bottom-4 right-4 z-50 flex flex-col gap-2`). Export it and mount it ONCE

- A `useToast()` hook (simple context-based, or a lightweight external store — your choice, but
